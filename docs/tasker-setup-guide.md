# TaskerによるSMS認証コード自動転送設定ガイド (Android)

スマートフォンの Android アプリ「Tasker」を使用して、ドコモ・バイクシェアから届くSMS認証コードを自動で抽出し、スクレイピングサーバーの Flask API へ転送するための具体的な設定手順です。

---

## 1. 事前準備

1. **Taskerアプリのインストール**
   - Google Play ストアから「Tasker」をインストールしておきます。
2. **送信先サーバーの外部公開**
   - スクレイピングサーバー（Flask）が動作しているPCのIPアドレス（例: `http://192.168.x.x:5000` またはグローバルドメイン）に対して、スマートフォンからネットワーク越しにアクセスできる必要があります。
   - ※同じWi-Fiルーターに接続している場合は、PCのローカルIPアドレス（`192.168.xx.xx`）で疎通可能です。
   - ※モバイル回線から送信する場合は、ポート開放やリバースプロキシ（ngrokなど）、またはCloudflare Tunnel等を用いて外部公開する必要があります。

---

## 2. プロファイル (トリガー) の作成

SMSを受信したときに動作を開始するトリガーを設定します。

1. Taskerを起動し、**「Profiles」** タブを開きます。
2. 右下の **「＋」** ボタンをタップし、**「Event」** を選択します。
3. カテゴリから **「Phone」** ＞ **「Received Text」** を選択します。
4. 設定画面（Event Edit）で以下のようにフィルターを設定します。
   - **Type**: `SMS`
   - **Sender**: （空欄のままでも良いですが、判明した場合はドコモ・バイクシェアのSMS送信元番号を指定します）
   - **Content**: `*認証コード*` または `*バイクシェア*` （アスタリスクで囲むことで、本文にそのワードが含まれる場合のみ起動させます）
5. 左上の **「←（戻る）」** ボタンをタップして確定します。

---

## 3. タスク (アクション) の作成

SMS受信時に実行する処理（認証コードの抽出とHTTP POST送信）を設定します。

### 3-1. 新規タスクの紐づけ
1. プロファイル確定後、ポップアップが表示されるので **「New Task ＋」** をタップします。
2. タスク名を入力します（例: `DBS SMS Transfer`）し、チェックマークをタップします。

### 3-2. アクション1：正規表現による認証コード抽出
本文（変数は `%SMSRB`）から、6桁（または4〜8桁）の数字を抽出します。

1. タスク編集画面で、右下の **「＋」** ボタンをタップします。
2. アクションの検索窓に **「Variable Search」** と入力するか、**「Variables」** ＞ **「Variable Search Replace」** を選択します。
3. アクションの設定を以下のように入力します。
   - **Variable**: `%SMSRB` （SMS本文が格納されているビルトイン変数）
   - **Search**: `\b\d{6}\b` （※6桁の数字を抽出する正規表現。4〜8桁など可変の場合は `\b\d{4,8}\b`）
   - **Multi-Line**: チェックを入れる
   - **Store Matches In**: `%code` （マッチした結果を格納する配列変数名。※パーセント記号付きで `%code` と入力します）
4. 左上の **「←（戻る）」** ボタンで保存します。

### 3-3. アクション2：HTTP POST リクエスト送信
抽出したコードをサーバーへ送信します。

1. タスク編集画面で、再度右下の **「＋」** ボタンをタップします。
2. アクションの検索窓に **「HTTP Request」** と入力するか、**「Net」** ＞ **「HTTP Request」** を選択します。
3. アクションの設定を以下のように入力します。
   - **Method**: `POST`
   - **URL**: `https://[あなたのサーバードメイン]/api/sms-code` (※`.env` に記載したホスト情報等を参照して設定してください。GitHub公開を防ぐため具体名は伏せています)
   - **Headers**:
     ```text
     Content-Type: application/json
     ```
   - **Body**: （※ダブルクォーテーションで囲んだJSON形式で、抽出した結果変数 `%code1` を指定します）
     ```json
     {"code": "%code1", "secret": "dbs-sms-secret"}
     ```
   - **Timeout**: `30`
4. 左上の **「←（戻る）」** ボタンで保存します。

---

## 4. 動作確認とデバッグ

### タスク単体での実行テスト
1. タスク編集画面の左下にある **「▶（再生）」** ボタンをタップすると、その場でタスクを実行できます。
2. テストの前に、一度変数 `%SMSRB` にダミーの本文を手動でセットしておくか、または HTTP Request の Body に直接数値を書いてテストします。
   - テスト用に一時的に `Body` を `{"code": "999999", "secret": "dbs-sms-secret"}` に変更して実行し、サーバー側で `999999` が登録されればAPI疎通は成功です。

### 最終有効化
1. Taskerのメイン画面に戻り、右上の **「チェックマーク（適用）」** ボタンをタップして設定を保存・有効化します。
2. 作成した Profile のスイッチが **「ON」** になっていることを確認してください。
